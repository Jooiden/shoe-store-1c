# Система автоматизации учета и управления для магазина обуви «Обувь»

## 📌 О проекте
Данный проект представляет собой автоматизированную систему на базе платформы **«1С:Предприятие 8.3»**, разработанную в рамках учебной практики. Система предназначена для оптимизации операционной деятельности магазина обуви, минимизации человеческого фактора, точного управления складскими запасами и обеспечения безопасности данных.

**Разработчик:** Сартбаев Жантай Болотбекович  
**Организация:** КГАПОУ «Камчатский политехнический техникум» (КПТ), 4 курс  

---

## 🎯 Цель и задачи проекта
**Цель:** Повышение эффективности операционной деятельности магазина через комплексную автоматизацию учета и исключение ручных рутинных операций.

**Основные задачи:**
1. Проектирование реляционной архитектуры базы данных.
2. Реализация многопользовательской ролевой модели доступа.
3. Создание модуля интеллектуального импорта нормативно-справочной информации (НСИ) из форматов Excel и JSON.
4. Программирование алгоритмов динамического ценообразования.
5. Реализация складского модуля, производственной цепочки и аналитической отчетности.

---

## 🛠️ Архитектура и функциональные возможности

### 1. Подсистема НСИ и Интеллектуальный импорт
Реализована обработка данных для автоматической загрузки справочников из внешних файлов (`.xlsx` и `.json`):
*   **Верификация данных:** Проверка по уникальному артикулу с защитой от дублирования.
*   **Динамическое создание:** Автоматическое создание связанных объектов (Контрагенты, Категории) «на лету» во время импорта.

### 2. Бизнес-логика и Динамическое ценообразование
Система автоматически формирует розничные цены на основе последней закупочной стоимости с учетом товарных групп:
*   **Гибкие наценки по категориям:** 
    *   Мужская — `42%`
    *   Женская — `40%`
    *   Детская — `31%`
    *   Пляжная / Домашняя — `22%`
*   **Прогрессивная система скидок:** Автоматический расчет скидки от `5%` до `20%` в зависимости от итоговой суммы чека при предъявлении карты лояльности.

### 3. Складская логистика и Производство (Make-to-Order)
Интегрирован полноценный контур управления запасами и позаказного производства:
*   **Учет сырья:** Контроль наличия материалов для выполнения индивидуальных заказов.
*   **Бизнес-процесс:** Проверка остатков $\rightarrow$ автоматический запуск пошива $\rightarrow$ оприходование готовой продукции на склад.
*   **Маршрутизация:** Логические связки между складскими операциями и торговым залом для обеспечения непрерывности процессов.

### 4. Ролевая модель доступа (Многопользовательский режим)
Права доступа разграничены между тремя основными категориями пользователей:
*   **Администратор:** Полный контроль над системой и конфигурацией.
*   **Менеджер:** Операционные функции (документы, продажи, импорт, производство) и формирование аналитических отчетов.
*   **Гость:** Просмотр каталога товаров, актуальных цен и формирование первичных заявок.

## 📊 Структура метаданных конфигурации

Основные объекты, реализованные в основной конфигурации системы:
*   **Справочники:** `Товары`, `Материалы`, `Клиенты`, `Контрагенты`, `Склады`, `Сотрудники`, `КатегорииТовара`, `Прайс` и др.
*   **Документы:** `ПриходнаяНакладная`, `РеализацияТоваров`, `ПеремещениеТоваров`, `ПриемЗаказов`, `ВыпускПродукции`.
*   **Регистры:** `ОстаткиНоменклатуры`, `Продажи` (накопления); `ЦеныНоменклатуры` (сведений).
*   **Отчеты:** `ОстаткиТоваров`, `Продажи`, `ОборотноСальдоваяВедомость` (на базе СКД).

## ⚙️ Инструкция по развертыванию

1. Создайте пустую информационную базу в программе «1С:Предприятие».
2. Откройте базу в режиме **Конфигуратор**.
3. В меню выберите: *Конфигурация -> Загрузить конфигурацию из файлов...* и укажите папку `src` из данного репозитория.
4. Обновите конфигурацию базы данных (клавиша `F7`).
